## 项目背景

### 概述

用户可通过全局快捷键 `⌘+Shift+Space` (macOS) / `Ctrl+Shift+Space` (Windows/Linux) 唤起/隐藏悬浮窗，也可在主窗口点击按钮开启悬浮窗。

### 核心需求

| 需求 | 描述 | 优先级 |
|------|------|--------|
| 全局快捷键 | `⌘+Shift+Space` (macOS) / `Ctrl+Shift+Space` (Windows/Linux) 唤起/隐藏悬浮窗 | P0 |
| 主窗口联动 | 主窗口提供按钮开启悬浮窗 | P1 |

## 任务列表

### 4.1 全局快捷键注册

**文件**: `packages/electron/src/main/shortcuts.ts`

```typescript
import { globalShortcut, dialog } from 'electron'

export function registerShortcuts(callback: () => void): boolean {
  const accelerator = process.platform === 'darwin'
    ? 'Cmd+Shift+Space'
    : 'Ctrl+Shift+Space'

  const success = globalShortcut.register(accelerator, callback)

  if (!success) {
    // MVP 阶段的优雅降级：弹窗提示冲突
    dialog.showErrorBox(
      '快捷键冲突',
      `无法注册全局快捷键 (${accelerator})，它可能被其他应用程序占用。请通过主窗口的按钮使用悬浮窗。`
    )
  }

  return success
}

export function unregisterShortcuts(): void {
  // 简单粗暴，完全符合 MVP，避免硬编码匹配错误
  globalShortcut.unregisterAll()
}
```

**关键点**:
- macOS: `⌘+Shift+Space`
- Windows/Linux: `Ctrl+Shift+Space`
- **注册时机**: 必须在 `app.whenReady()` 回调中调用，否则会报错或静默失败
- **注销时机**: 在 `app.on('will-quit', unregisterShortcuts)` 中调用
- 注册失败时通过 Dialog 告知用户，避免用户困惑
- 使用 `unregisterAll()` 简化注销逻辑，避免硬编码匹配错误

**验收标准**：
- [ ] 快捷键可正常注册
- [ ] 快捷键可唤起/隐藏悬浮窗
- [ ] 应用退出时快捷键注销
- [ ] 注册失败时显示冲突提示对话框

## 快捷键行为说明

### 悬浮窗状态机

```
悬浮窗状态：
1. 不存在 → 按快捷键 → 创建并显示
2. 存在且可见且聚焦 → 按快捷键 → 隐藏
3. 存在但被遮挡/未聚焦 → 按快捷键 → 置顶显示并聚焦
```

### toggleFloatingWindow 逻辑

> ⚠️ **关键修复**: 必须增加 `isDestroyed()` 校验，防止调用已销毁的窗口实例导致崩溃。

```typescript
function toggleFloatingWindow() {
  // 修复：检查窗口是否已被销毁
  // floatingWindow 变量可能存在但底层 WebContents 已被销毁
  if (floatingWindow && !floatingWindow.isDestroyed()) {
    // 只有既可见又有焦点时才隐藏
    // 否则（被其他窗口遮挡或未聚焦）应该置顶显示
    if (floatingWindow.isVisible() && floatingWindow.isFocused()) {
      floatingWindow.hide()
    } else {
      floatingWindow.show()
      floatingWindow.focus()
    }
  } else {
    floatingWindow = createFloatingWindow(isDev)

    // 关键：监听 closed 事件，清理引用
    // 防止窗口被强制关闭后 floatingWindow 变量仍持有已销毁的引用
    floatingWindow.on('closed', () => {
      floatingWindow = null
    })
  }
}
```

**防崩溃关键点**:
- 调用 `isVisible()` / `isFocused()` 前必须先检查 `!isDestroyed()`
- 在窗口的 `closed` 事件中将引用置空
